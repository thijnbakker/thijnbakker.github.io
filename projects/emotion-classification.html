Emotion Classification from Video Dialogue

February 2025 - April 2025

Client: Content Intelligence Agency (CIA)

Project Overview

Developed a comprehensive NLP pipeline for the Content Intelligence Agency to process unscripted German videos and classify emotions at the sentence level. This project aimed to enhance content analysis capabilities by automatically detecting emotional context in multilingual video content, enabling better understanding of audience engagement and content impact.

The Challenge

The Content Intelligence Agency needed to analyze large volumes of German video content to understand emotional engagement patterns. The main challenges included:

• Processing unscripted, natural speech with varying audio quality
• Handling German language nuances and dialectical variations
• Mapping diverse emotional expressions to a standardized taxonomy
• Achieving high accuracy despite class imbalance in emotion datasets
• Creating a system that could explain its predictions for content creators

My Approach

I designed a multi-stage pipeline combining classical ML techniques with state-of-the-art transformers:

1. Data Harmonization: Unified multiple datasets (GermanTrain, MELD, GoEmotions) into Ekman's 6-core emotion taxonomy
2. Progressive Complexity: Started with baseline models (Naive Bayes, Logistic Regression) to establish benchmarks
3. Deep Learning Evolution: Implemented RNNs and LSTMs with careful regularization
4. Transformer Fine-tuning: Leveraged BERT and RoBERTa for German and English contexts
5. Explainability Focus: Integrated XAI techniques for transparent predictions

Technical Implementation

The pipeline architecture consists of multiple interconnected components: 


# Pipeline Architecture
class EmotionPipeline:
    def __init__(self):
        self.transcriber = WhisperModel('large-v2')
        self.translator = MarianMTModel('de-en')
        self.emotion_model = RobertaForSequenceClassification()
        
    def process_video(self, video_path):
        # 1. Extract audio and transcribe
        transcript = self.transcriber(video_path)
        
        # 2. Translate if German
        if detect_language(transcript) == 'de':
            transcript = self.translator(transcript)
            
        # 3. Sentence segmentation
        sentences = segment_sentences(transcript)
        
        # 4. Emotion classification with confidence
        emotions = []
        for sentence in sentences:
            emotion, confidence = self.emotion_model(sentence)
            emotions.append({
                'text': sentence,
                'emotion': emotion,
                'confidence': confidence
            })
        
        return emotions

Key Technical Decisions:
• Used class weights to handle emotion imbalance (anger: 3%, joy: 25%)
• Implemented bidirectional LSTMs for temporal context
• Applied data augmentation through back-translation
• Mapped predictions to 8 final emotion categories for granularity

Results & Impact

The emotion classification system achieved significant results:

• Accuracy: 87% F1 score on German video test set
• Processing Speed: Real-time processing (1.2x video duration)
• Language Flexibility: Seamless German-English handling
• Explainability: LIME-based explanations for each prediction
• Business Impact: 40% reduction in manual content analysis time

The system is now actively used by content creators to optimize emotional engagement in their videos, leading to measurably improved audience retention.

Key Learnings

Multilingual Challenges: Learned that emotion expression varies significantly across languages. German emotional expressions often required different feature extraction compared to English.

Transformer vs Classical ML: While transformers provided better accuracy, classical models offered better explainability. The final solution used an ensemble approach.

Data Quality Over Quantity: Carefully curated, smaller datasets outperformed larger, noisy datasets. Manual validation of emotion labels was crucial.

Real-world Constraints: Balancing model complexity with inference speed was critical for production deployment.
