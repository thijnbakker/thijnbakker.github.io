DriveSafe - AI Traffic Sign Detection System

August 2023 - November 2023

Client: Innovation Square - Breda University of Applied Sciences

Individual Project

Project Overview

DriveSafe represents my first major AI project during my first year at BUaS - a comprehensive traffic sign detection system that combines deep learning with thoughtful user experience design. This project tackled the challenge of creating a reliable AI system that can identify traffic signs in real-world conditions while designing an intuitive interface that enhances rather than distracts drivers.

The Challenge

The project addressed several interconnected challenges in the automotive AI space:

• Technical Challenge: Developing a deep learning model capable of accurately classifying traffic signs in diverse lighting and weather conditions
• Business Challenge: Identifying the market need and stakeholder requirements for traffic sign detection systems
• UX Challenge: Designing an interface that provides critical information without causing driver distraction
• Performance Challenge: Achieving 90%+ accuracy through iterative model improvement
• Evaluation Challenge: Conducting rigorous A/B testing to validate design decisions

My Approach

This project followed a comprehensive methodology spanning business understanding, technical development, and user experience design:

1. Market Research & Stakeholder Analysis: Identified car manufacturers as primary stakeholders, analyzed the competitive landscape, and defined the value proposition for traffic safety systems
2. Deep Learning Development: Built multiple CNN architectures, from simple custom models to transfer learning with MobileNet
3. Iterative Model Improvement: Conducted 4 major iterations, progressively improving from 27.5% to 90.8% accuracy
4. UI/UX Design: Created two wireframe prototypes in Figma and conducted extensive user testing
5. Statistical Validation: Performed rigorous A/B testing with statistical significance analysis

Technical Implementation

The final solution emerged through systematic iteration and experimentation with multiple architectures: 


# DriveSafe Traffic Sign Detection - Final Model Architecture
import tensorflow as tf
from tensorflow.keras.applications import MobileNet
from tensorflow.keras.layers import Dense, GlobalAveragePooling2D
from tensorflow.keras.models import Model

class DriveSafeClassifier:
    def __init__(self, num_classes=4):
        # Best performing model: Transfer Learning with MobileNet
        self.base_model = MobileNet(
            include_top=False,
            weights='imagenet',
            input_shape=(128, 128, 3)
        )
        
        # Custom classification head
        x = GlobalAveragePooling2D()(self.base_model.output)
        x = Dense(128, activation='relu')(x)
        predictions = Dense(num_classes, activation='softmax')(x)
        
        self.model = Model(inputs=self.base_model.input, outputs=predictions)
        
        # Fine-tuning strategy
        self.base_model.trainable = False  # Freeze base layers initially
        
    def compile_model(self):
        self.model.compile(
            optimizer='adam',
            loss='categorical_crossentropy',
            metrics=['accuracy']
        )
        
    def progressive_training(self, train_data, val_data):
        # Phase 1: Train classifier head
        history1 = self.model.fit(
            train_data,
            validation_data=val_data,
            epochs=10,
            callbacks=[tf.keras.callbacks.EarlyStopping(patience=3)]
        )
        
        # Phase 2: Fine-tune entire model
        self.base_model.trainable = True
        
        # Lower learning rate for fine-tuning
        self.model.compile(
            optimizer=tf.keras.optimizers.Adam(1e-5),
            loss='categorical_crossentropy',
            metrics=['accuracy']
        )
        
        history2 = self.model.fit(
            train_data,
            validation_data=val_data,
            epochs=5
        )
        
        return history1, history2

User Experience Design & A/B Testing

The project included comprehensive UX research and statistical validation through A/B testing:

Design Process:
• Created two distinct wireframe prototypes (A and B) using Figma
• Wireframe A: Minimal, basic interface with essential functionality
• Wireframe B: Enhanced design with improved navigation and visual appeal
• Conducted user testing sessions with think-aloud protocols
• Gathered quantitative feedback using 7-point Likert scales

Statistical Analysis Results:
• Conducted Independent Samples T-tests with normality and homogeneity checks
• Wireframe B significantly outperformed Wireframe A (p < 0.05)
• Design quality: B received no scores below 3, A received multiple 1s
• Navigation: B received five 7-point scores, A received zero
• User experience: All participants enjoyed Wireframe B vs only 3 for A
• Understanding: B had zero scores below 5, A showed wide variation

Key UX Insights:
• Visual design significantly impacts perceived usability
• Navigation clarity is crucial for driver-assistance interfaces
• Statistical validation provided objective evidence for design decisions
• User feedback directly informed the final prototype development

Results & Impact

DriveSafe achieved significant milestones across technical, business, and design dimensions:

• Technical Achievement: 90.8% classification accuracy using MobileNet transfer learning
• Model Performance: F1 scores ranging from 27% to 94% across different traffic sign classes
• Class-Specific Success: Excellent performance on certain sign types (95% precision for some classes)
• Design Validation: Statistically significant UX improvements through A/B testing
• Academic Excellence: Comprehensive project covering full AI product development lifecycle
• Business Understanding: Detailed stakeholder analysis and market research
• Documentation: Complete project presentation and demo video created

This first-year project established my foundation in AI development, demonstrating the ability to combine technical machine learning skills with business acumen and user-centered design principles.

Key Learnings

Transfer Learning Power: The dramatic performance improvement from 52% to 90.8% when switching to MobileNet demonstrated the value of pre-trained models. Custom architectures couldn't match the feature representations learned on ImageNet.

Class Imbalance Reality: Real-world datasets present significant challenges. Some traffic sign classes had much lower performance (25% precision), highlighting the need for careful data curation and balanced evaluation metrics.

Statistical Rigor in UX: Conducting proper statistical tests (Shapiro-Wilk, Levene's test, Independent T-tests) provided objective evidence for design decisions. Wireframe B's superiority wasn't just subjective - it was statistically significant.

Holistic Product Development: This project taught me that successful AI systems require more than just model accuracy. Market research, stakeholder analysis, user experience design, and statistical validation are equally crucial.

Iterative Improvement: The journey from 27.5% to 90.8% accuracy demonstrated the importance of systematic experimentation and the compound benefits of incremental improvements.

Business Context & Market Analysis

The project included comprehensive business analysis identifying the commercial potential:

Primary Stakeholders: Car manufacturers emerged as the key stakeholder with the highest power and interest in traffic sign detection technology for ADAS (Advanced Driver Assistance Systems) integration.

Value Proposition:
• Reduced traffic violations and fines for commercial fleets
• Enhanced safety for delivery companies and professional drivers
• Competitive advantage for automotive manufacturers
• Regulatory compliance support for autonomous vehicle development

Market Opportunity: Analysis revealed significant demand from government transportation agencies, insurance companies, and end users, with car manufacturers positioned as the ideal integration partner.
