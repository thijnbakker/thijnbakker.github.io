Project Overview

Developed the "Risk Awareness" application for ANWB Safe Driving to help drivers identify high-risk areas before entering them. The application uses GPS tracking and machine learning to provide real-time risk scores based on historical incident data, driving behavior patterns, and environmental conditions. When users enter a high-risk zone, their smartphone or smartwatch vibrates with a unique pattern to alert them, promoting safer driving awareness.

The Challenge

Modern drivers often lack real-time awareness of high-risk areas, leading to potentially hazardous situations. The project aimed to address several key challenges:

‚Ä¢ Risk Prediction Accuracy: Building reliable ML models to predict accident probability from telematics data
‚Ä¢ Data Integration: Combining ANWB incident data with KNMI weather data for comprehensive risk assessment
‚Ä¢ Real-time Processing: Creating a system capable of processing GPS coordinates and providing instant risk scores
‚Ä¢ Legal Compliance: Ensuring GDPR and EU AI Act compliance for a high-risk AI system
‚Ä¢ User Experience: Designing intuitive alerts that inform without distracting drivers
‚Ä¢ Mobile Deployment: Creating a production-ready mobile application using FlutterFlow

Technical Implementation

The system processes historical incident data to predict accident probability in specific geographic areas: 


                    # Time Series Analysis Pipeline
                    import torch
                    import torch.nn as nn
                    from sklearn.ensemble import GradientBoostingClassifier
                    
                    class DrivingBehaviorAnalyzer:
                        def __init__(self):
                            self.rnn_model = self.build_rnn()
                            self.gbm = GradientBoostingClassifier(n_estimators=100)
                            self.anomaly_threshold = None
                            
                        def build_rnn(self):
                            return nn.Sequential(
                                nn.LSTM(input_size=15, hidden_size=128, num_layers=2, 
                                       batch_first=True, dropout=0.2),
                                nn.Linear(128, 64),
                                nn.ReLU(),
                                nn.Linear(64, 5)  # 5 risk categories
                            )
                        
                        def extract_features(self, telematics_data):
                            features = {
                                'harsh_braking': self.detect_harsh_events(
                                    telematics_data['acceleration'], threshold=-4.5),
                                'rapid_acceleration': self.detect_harsh_events(
                                    telematics_data['acceleration'], threshold=3.5),
                                'cornering_speed': self.analyze_cornering(
                                    telematics_data['gyro'], telematics_data['speed']),
                                'time_of_day_risk': self.temporal_risk_score(
                                    telematics_data['timestamp']),
                                'location_risk': self.spatial_risk_score(
                                    telematics_data['gps'])
                            }
                            return features
                        
                        def predict_risk(self, features):
                            # Ensemble prediction
                            rnn_pred = self.rnn_model(features['sequences'])
                            gbm_pred = self.gbm.predict_proba(features['aggregated'])
                            
                            # Weighted average
                            final_risk = 0.6 * rnn_pred + 0.4 * gbm_pred
                            return final_risk, self.generate_explanation(features, final_risk)

Key Learnings

Model Selection Importance: The comprehensive comparison revealed that Gradient Boosting significantly outperformed Decision Trees due to better generalization. Decision Trees showed >10% overfitting while Gradient Boosting maintained <1% overfitting, making it the clear choice for production deployment.

Data Quality Impact: Extensive preprocessing was crucial - removing duplicates, handling missing values, and outlier detection improved model performance by approximately 15%. The integration of weather data provided additional predictive power for risk assessment.

Legal Complexity in AI Systems: Building a high-risk AI system required extensive compliance work. GDPR Article 7 (user consent), Article 15 & 21 (data access/control), Article 17 (data deletion), and Article 22 (explanation rights) all needed implementation. EU AI Act Annex III, IV, and XII requirements added significant development overhead.

Real-world Deployment Challenges: Moving from prototype to mobile application revealed the complexity of real-time GPS processing, battery optimization, and user experience design. FlutterFlow provided rapid development capabilities but required careful optimization for production use.

Team Collaboration in Data Science: Working with 4 team members on a complex ML project required clear role definition and strong communication. Version control, code documentation, and standardized evaluation metrics were essential for successful collaboration.
